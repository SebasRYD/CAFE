Arquitectura del Sistema

Una visión general de cómo funciona el sistema de gestión de cafetería

Flujo Principal del Sistema

El sistema está diseñado con una arquitectura modular que permite la gestión integral de una cafetería. El flujo principal comienza en la clase Main.java que presenta un menú interactivo con las siguientes opciones:

  1. Gestión de Mesas: Control de mesas, comensales y pedidos
  2. Gestión de Productos: Catálogo completo del menú
  3. Gestión de Tareas: Sistema de asignación y seguimiento de tareas
  4. Gestión de Empleados: Administración del personal
  5. Análisis de Ventas: Reportes y estadísticas
  6. Demostración: Ejecución automática de funciones demo

Cada módulo funciona de manera independiente pero se integra perfectamente con los demás, permitiendo un flujo de información coherente a través de toda la aplicación.

Módulos del Sistema

Descripción detallada de cada componente del sistema

GestionMesas.java

Este módulo se encarga de la gestión completa de las mesas del establecimiento. Sus funciones principales incluyen:

Funcionalidades:

  • Creación y eliminación de mesas
  • Gestión de comensales por mesa
  • Registro de pedidos con sistema FIFO
  • Procesamiento de pedidos pendientes
  • Generación de tickets al cerrar una mesa
  • Almacenamiento de tickets en archivos de texto

Estructuras de Datos Utilizadas:

  • HashMap: Para almacenar y acceder rápidamente a las mesas por número
  • Queue (LinkedList): Para gestionar pedidos en orden FIFO
  • LocalDateTime: Para registrar timestamps de pedidos

MenuAlimentos.java

Módulo responsable del catálogo de productos disponibles en la cafetería.

Funcionalidades:

  • Carga del menú desde archivo CSV
  • Organización de productos por categorías
  • Búsqueda de productos por ID o nombre
  • Gestión de pedidos individuales
  • Generación de tickets con formato profesional

Estructuras de Datos Utilizadas:

  • Array: Para almacenar los productos del menú
  • LinkedHashMap: Para mantener el orden de las categorías
  • Queue (LinkedList): Para los pedidos en los tickets

GestionTareas.java

Sistema de gestión de tareas con prioridades para la organización del trabajo del personal.

Funcionalidades:

  • Creación de tareas con prioridades (1-5)
  • Procesamiento por prioridad (PriorityQueue)
  • Búsqueda recursiva de tareas
  • Distribución de tareas usando "Divide y Vencerás"
  • Cálculo de tiempo total de tareas pendientes
  • Asignación automática a empleados por departamento

Estructuras de Datos Utilizadas:

  • PriorityQueue: Para gestionar tareas por prioridad
  • Recursión: Para búsquedas y cálculos
  • Algoritmo Divide y Vencerás: Para distribución de tareas

GestionEmpleados.java

Sistema de administración de empleados utilizando un árbol binario de búsqueda.

Funcionalidades:

  • Inserción y eliminación de empleados
  • Búsqueda eficiente por ID
  • Recorrido inorden para mostrar empleados ordenados
  • Cálculo recursivo de nómina total
  • Persistencia en archivo CSV
  • Filtrado por departamento

Estructuras de Datos Utilizadas:

  • Árbol Binario de Búsqueda (BST): Para almacenamiento y búsqueda eficiente
  • Recursión: Para operaciones del árbol
  • ArrayList: Para manejo de listas de empleados

EstadisticasVentas.java

Módulo de análisis de ventas con algoritmos de ordenamiento y búsqueda.

Funcionalidades:

  • Lectura y análisis de tickets guardados
  • Ordenamiento con Quicksort recursivo
  • Búsqueda binaria en datos ordenados
  • Análisis de tendencias de productos
  • Cálculo de ventas totales del día

Estructuras de Datos Utilizadas:

  • List/ArrayList: Para manejo de datos de ventas
  • Quicksort: Para ordenamiento eficiente
  • Búsqueda Binaria: Para búsqueda rápida en datos ordenados
  • HashMap: Para análisis de productos más pedidos

Estructuras de Datos Implementadas

Análisis de las estructuras de datos utilizadas en el sistema

HashMap

Uso: Almacenamiento de mesas por número para acceso rápido O(1)

Implementación: En GestionMesas para el mapa de mesas activas

Queue (FIFO)

Uso: Gestión de pedidos en orden de llegada

Implementación: LinkedList para pedidos pendientes, procesados y cancelados

PriorityQueue

Uso: Gestión de tareas por prioridad

Implementación: En GestionTareas con comparador personalizado

Árbol Binario de Búsqueda

Uso: Almacenamiento y búsqueda eficiente de empleados

Implementación: BST personalizado en GestionEmpleados

List/ArrayList

Uso: Almacenamiento de elementos en secuencia

Implementación: Múltiples usos en todo el sistema para listas de datos

Algoritmos de Ordenamiento

Uso: Ordenamiento eficiente de datos

Implementación: Quicksort recursivo en EstadisticasVentas

Implementación de Recursión

El sistema hace un uso extensivo de la recursión para resolver problemas complejos de manera elegante:

  • GestionTareas: Búsqueda recursiva, cálculo de tiempo total, distribución de tareas
  • GestionEmpleados: Operaciones del árbol binario (inserción, búsqueda, eliminación)
  • EstadisticasVentas: Quicksort recursivo, búsqueda binaria recursiva
